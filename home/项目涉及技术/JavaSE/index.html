一、基础

​ Java编码规范

  • 严格区分大小写
  • 源文件以“ .java” 为后缀，基本组成单元为 class
  • 一个 Java 源文件只能由一个 public 类，其他非 public 不限。当存在 public 类时，源文件名称必须与该类同名区分大小写
  • 程序主入口为 main 方法，标准格式为: public static void main(Stringp[ ] args ){ }
  • 类内部的方法代码以语句为最小单位，每个语句以 " ; " 为分号结束

1、标识符

1）释义

  标识符就是用于给程序中的变量、类、方法命名的符号。标识符可以由编程人员自由定义，但需要遵循一定的命名规则。

2）命名规则

  可以由字母、数字、下划线 ( __ )、美元符号 ( $ ) 组成。

  必须以字母、下划线或美元符号开头，不能以数字开头。

  不能指定关键字。

  长度没有限制，但定义时一般以见名知义为原则。

  使用驼峰命名，类名首字母均大写，常量全部大写；方法名、属性名的单词首字母小写中间的单词首字母大写。

2、关键字

  Java语言中用以特殊用途而预占用的单词或标识符称之为关键字，Java中的关键字均为小写。

  class、extends、implements、interface、package、break、case、continue、do、if、else、for等。

3、常量、变量及作用域

  常量也可称为字面值，即字面显示的值，其本身不会发生变化。

  一种 flnal修饰的变量也称为常量。

  常量在程序运行过程中其值不能改变。变量是用于存储可变数据的容器，存储一定格式的可变数据。

  命名规范不一样。默认常量为全大写，而变量为驼峰。

  修饰符不同。一般常量都会用final修饰。变量不会，而且为了节省内存，常量都会同时使用 static 修饰 。

  变量是程序中最基本的存储单元，由作用域、变量类型、变量名组合。

  变量分类：局部变量、成员变量、类变量

​ 局部变量（本地变量）：

  方法的形式参数以及在方法中定义的变量。

  格式：type name = value

  作用范围：

  形参：在方法体中任何位置都可以访问

  方法中定义变量：从定义处开始，直到所在代码块结束

  生命周期：

    出生：运行到创建变量语句时

    死亡：超过其作用范围

​ 成员变量（实例变量，属性）：

  成员变量就是类中的属性。当 new 对象的时候，每个对象都有一份属性。一个对象中的属性就是成员变量。

  格式：访问修饰符 type name = value

  作用范围：在类内部，任何地方都可以访问成员变量

  声明周期：

    出生：new 对象的时候，开辟内存空间

    死亡：堆内存地址没有引用，变成垃圾，被垃圾回收器回收后

​ 类变量（静态属性）：

  被 static 修饰的属性。

  格式：访问修饰符 static type name = value

  作用范围：在类变量定义之后

  声明周期：

    出生：类加载时，类变量就分配内存空间

    死亡：JVM退出

4、数据类型

  数据类型是为了把数据分成所需内存大小不同的数据，编程的时候需要用大的数据才申请大的内存，这样可以充分利用内存。

  Java语言是一种强类型语言，要求所有变量先声明后使用。目的就是为了保证变量或者表达式在编译时就能够确定其类型，并为其分配相应的内存。

数据类型

4.1 整型

​ 整型：byte、short、int、long

​ 取值范围：

  Java语言整型默认为int类型，如果要声明成long类型在变量值后加入L，如: long i = 123L

类型 占用存储空间字节数 数值范围
byte 1 -128~127
short 2 -2^15~2^15-1
int 4 -2^31~2^31-1
long 8 -2^63~2^63-1

​ Java整型常量的三种表示方法：

十进制,例如: 123=1*100+2*10+3=123
八进制，例如: 012=1*8+2*1=10
十六进制，例如: 0x32=3*16+2*1=50

4.2 浮点型

​ 浮点型：float、double

​ 取值范围：

  浮点声明默认是double类型，若要是float，需要在尾部添加f或者F。如：float f = 3.14F

类型 占用存储空间字节数 数据范围
float 4 -3.4E38~3.4E38
double 8 -1.7E308~1.7E308

4.3 布尔型

​ 布尔型：true、false

  适用于逻辑运算，用于程序流程控制

  值允许取值为true、false，不可以任何其他形式代替

4.4 字符型

  通常意义上的字符，是用单个引号括起来的单个字符

如 'a','2','A'

  多个字符组合在一起，称为字符串

如 "Hello Tom"

  转义字符，用 '\n' 来表示，转变之前依然是个字符

如 '\n'

4.5 基本类型转换

  • 布尔类型不能与其他类型做任何转换
  • 数值型、字符型可以相互转换，规则如下
  • 容量小的自动转换成表数范围容量大的，但 byte,short,char 之间不能相互转换，均转化成int做计算
  • 容量大小排序为: byte,char,short < int < long < float < double
  • 容量大转换为容量小的，要加强制转换符，此时容易丢失精度
  • 多类型混合运算时，java会自动按照最大的类型转换成统一类型后再进行计算，即先类型转换后计算

6、流程控制结构

  流程控制结构是指以某种顺序执行的一系列动作，用于解决某个问题。程序可以通过控制语句来对程序实现选择、循环、转向和返回等流程控制。

  流程控制结构包括：顺序结构、分支结构、循环结构。

6.1 顺序结构

  顺序结构程序就是按语句出现的先后顺序执行的程序结构。计算机按顺序逐条执行语句，当一条语句执行完毕，自动转到下一条语句。

顺序结构

6.1.1 分支结构

1、分支结构又称为选择结构。当程序执行到控制分支语句时，首先判断条件，根据条件表达式的值选择相应的语句执行（放弃另一部分语句的执行）。

2、分支结构包括单分支、双分支和多分支三种形式。

分支结构

6.1.2 语法

if(布尔表达式){ 
    语句块; 
}else if(布尔表达式){ 
    语句块; 
}else{ 
    语句块; 
}

 switch (表达式) {
    case 值1:
        语句;
        break;
    case 值2:
        语句;
        break;
    default:
        语句;
        break;
 }
  • 表达式的值只能为：char、byte、short、int类型、String、enum类型,其它类型均非法
  • break语句可以省略,但会出现switch穿透
  • default语句也可以省略,一般不建议省略,并且放置在最后

6.2 循环结构

1、当程序执行到循环控制语句时，根据循环判定条件对一组语句重复执行多次。

2、循环结构的三个要素：循环变量、循环体和循环终止条件。

3、while、do…while、for三种循环。

循环结构

6.2.1 while语法

while(布尔表达式){ 
循环体; //一条或多条语句 
}

6.2.2 do…while语句

do { 
   循环体; 
}while(条件判断表达式);

6.2.3 for语句

for(表达式1;表达式2;表达式3) { 
    循环体; //一条或多条语句 
}

6.2.4 break语句

强制退出某个循环体，结束循环语句的执行

6.2.5 continue语句

终止某次循环过程，跳过continue语下方未执行的代码，开始下一步循环过程

6.2.6 流程控制语句总结

条件判断语句:if语句、switch语句。
循环执行语句:do while语句、while语句、for语句。
跳转语句:break语句、continue语句、return语句。
控制语句类型 关键字 作用
选择结构语句 if、if else、else if、switch 通过开关机制，选择要执行的代码
循环结构语句 for、while、do while 通过循序机制，反复执行相同的代码段
改变语句执行序 break、continue 通过打断或继续机制，改变当前工码的执行顺序

三种循环结构的异同：

  • 用while和do…while循环时,循环变量在循环体之前初始化，而for循环一般在语句1进行初始化。
  • while 循环和for循环都是先判断表达式,后执行循环体;而do…while循环是先执行循环体后判断表达式。 也就是说do…while的循环体最少被执行一次，而while循环和for就可能一次都不执行。
  • 这三种循环都可以用break语句跳出循环,用continue语句结束本次循环。

  • 提高程序开发效率

7.2.5 递归方法

  程序自身调用自身的编程技巧称为递归

递归四个特性：

  • 必须有可最终达到的终止条件，否则程序将陷入无穷循环
  • 子问题在规模上比原问题小，或更接近终止条件
  • 子问题可通过再次递归调用求解或因满足终止条件而直接求解
  • 子问题的解应能组合为整个问题的解

三、数组

3.1、数组

  数组是在程序设计中，为了处理方便， 把具有相同类型的若干元素按无序的形式组织起来的一种形式。这些无序排列的同类数据元素的集合称为数组。

3.2、一维数组

3.2.1 一维数组概述

一维数组

​

一维数组声明

两种声明格式：
元素类型[]  变量名称  例如：int[]  intArray;
元素类型  变量名称[]  例如：int  intArray[];
一行中允许声明多个数组
int[]  array1,array2,array3

一维数组初始化

数组初始化的过程就是为数据分配内存空间的过程。

静态初始化：

声明数组时，对数组初始化，数组的大小会根据初始值的个数确定长度。
例如：
public class Demo1 {
    public static void main(String[] args) {
        String[] arr1 = {"a","b","c","d"};
    }
}

动态初始化：

声明数组时，指定数组的长度即可。
例如：
public class Demo1 {
    public static void main(String[] args) {
        int[] arr1 = new int[10];
    }
}

3.5、数组排序

3.5.1 冒泡排序

  冒泡排序(Bubble Sort)，是一种计算机科学领域的较简单的排序算法。

它重复地走访过要排序的数列，一次比较两个元素，如果他们的顺序错误就把他们交换过来。走访数列的工作是重复地进行直到没有再需要交换，也就是说该数列已经排序完成。这个算法的名字由来是因为越大的元素会经由交换慢慢 “浮”到数列的顶端，故名 “冒泡”。

【算法原理】

  1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
  2. 对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对。在这一点，最后的元素应该会是最大的数。
  3. 针对所有的元素重复以上的步骤，除了最后一个。
  4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

举例：

public class Demo1 {
    public static void main(String[] args) {
        int[] arr1 = {10,8,9,5,3,6,7,1,2};
        int temp = 0;
        for (int i = 0; i < arr1.length - 1; i++) {
            for (int j = 0; j < arr1.length - 1 - i; j++) {
                if (arr1[j] > arr1[j+1]){
                    temp = arr1[j+1];
                    arr1[j+1] = arr1[j];
                    arr1[j] = temp;
                }
            }
        }
        for (int i : arr1) {
            System.out.print(" " + i);
        }
    }
}

3.5.2 选择排序

选择排序(Selection Sort)是一种简单直观的排序算法。它每一次从待排序的数据元素中选出最小(或最大)的一个元素，存放在序列的起始位置，直到全部待排序的数据元素排完。

【算法原理】

  1. 在未排序序列中找到最小元素，存放到排序序列的起始位置
  2. 再从剩余未排序元素中继续寻找最小元素，然后放到排序序列末尾。
  3. 以此类推，直到所有元素均排序完毕。

举例：

public class Demo2 {
    public static void main(String[] args) {
        int[] arr1 = {10,8,9,5,3,6,7,1,2};
        int temp = 0;
        for (int i = 0; i < arr1.length - 1; i++) {
            for (int j = i + 1; j < arr1.length ; j++) {
                if (arr1[i] > arr1[j]){
                    temp = arr1[j];
                    arr1[j] = arr1[i];
                    arr1[i] = temp;
                }
            }
        }
        for (int i : arr1) {
            System.out.print(" " + i);
        }
    }
}

或者：

public class Demo3 {
    public static void main(String[] args) {
        int[] arr1 = {10,8,9,5,3,6,7,1,2};
        int temp = 0;
        for (int i = 0; i < arr1.length; i++) {
            int k = i; //待确定的位置
            //选择出应该在第i个位置的数
            for (int j = arr1.length - 1; j > i; j--){
                if (arr1[j] < arr1[k]){
                    k = j;
                }
            }
            //交换两个数
            temp = arr1[i];
            arr1[i] = arr1[k];
            arr1[k] = temp;
        }
    }
}

3.5.3 快速排序

  快速排序是一个既不浪费空间，速度快的排序方法。

【算法原理】

快速排序

​

  1. 取数组最左为左点(15)，取数组最右边为右点(16)，取数组中间值为基准点（即将数据分为两组的中间值点10）
  2. 左点向右走，直到大于中间值(15>10)，此时右点向左走，直到小于中间值(8<10)，交换左右点的值
  3. 左点向走，直到大于中间值，此时右点向左走，直到小于中间值，第一轮结束
  4. 对中间值左右两变两个数组调用自身排序方法(递归)，将左右两个数组排序

举例：

public class Demo4 {
    public static void main(String[] args) {
        int[] arr1 = {15,7,10,8,16};
        quickSort(arr1);
        for (int i : arr1) {
            System.out.print(" " + i);
        }
    }

    //快排方法
    public static void quickSort(int[] data){
        subSort(data, 0, data.length - 1);
    }

    //主要排序逻辑
    private static void subSort(int[] data,int start,int end){
        if (start < end){ //递归终止条件
            int base = data[(start + end)%2];
            int low = start;
            int high = end + 1;

            while (true){ // 循环找出左>base，右<base的值
                while (low < end && data[++low] <= base); // 找出左边第一个大于base的值
                while (high > start && data[--high] >= base); // 找出右边第一个小于base的值
                if (low < high){
                    swap(data,low,high); // 更换左右两值
                }else {
                    break; // 交界线
                }
            }
            swap(data,start,high); // 将第一个值和high交换

            // 左右分为两个数组，以base为分界线，分别递归调用
            subSort(data,start, high - 1);
            subSort(data,high + 1,end);
        }
    }

    //更换值的方法
    private static void swap(int[] data, int left, int right){
        int temp = 0;
        temp = data[right];
        data[right] = data[left];
        data[left] = temp;
    }
}

3.5.4 归并排序

  归并排序(merge-sort)是利用归并的思想实现的排序方法，该算法采用的是经典的分治(divide-and-conquer)策略（分治法将问题分(divide)成一些小的问题然后递归求解，而治(conquer)的阶段将分的阶段得到的结果合并在一起，即分而治之）。

算法原理：

归并排序

​

  不断的将大的数组分成两个小数组，直到U币能拆分为止，即形成了单个值。此时使用合并的排序思想对已经有序的数据进行合并，合并为一个大的数据 ，不断重复此过程，直到最终所有的数据合并到一个数组为止。

举例：

public class Demo5 {
    public static void main(String[] args) {
        int arr[] = {5,3,7,7,4,1,8,6};
        int[] add = new int[arr.length];

        mergeSort(arr,0, arr.length - 1,add);

        System.out.println(Arrays.toString(arr));
    }
    public static void mergeSort(int[] arr,int left,int right,int[] temp){
        //取出中间值的索引
        int mid = (left+right)/2;
        if (left < right){
            // 一分为二
            mergeSort(arr,left,mid,temp); //左边递归调用
            mergeSort(arr,mid + 1,right,temp); //右边递归调用
            merge(arr,left,mid,right,temp); //分解到最散的时候开始向上排序合并
        }
    }
    private static void merge(int[] arr,int left,int mid,int right,int[] temp){
        int i = left;
        int j = mid + 1;
        int t = 0;

        //将小的值放入临时数组左边 => 排序
        while(i <= mid && j <= right){
            if (arr[i] < arr[j]){
                temp[t] = arr[i++];
            }else{
                temp[t] = arr[j++];
            }
            t++;
        }
        //将剩余的数依次放入到temp
        while(i <= mid){
            temp[t] = arr[i];
            t++;
            i++;
        }
        while (j <= right){
            temp[t] = arr[j];
            t++;
            j++;
        }
        //将临时数组的值添加到原始数组
        int n = 0;
        int index = left;
        while (index <= right){
            arr[index] = temp[n];
            n++;
            index++;
        }
    }
}
